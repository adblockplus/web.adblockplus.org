title=Detailed changelog for Adblock Plus 1.3.5

{{s1 The following lists the major changes compared to Adblock Plus 1.3.3.}}

{{s2 Features}}

  • {{s3 Firefox Mobile 4 is now supported, support for Firefox Mobile 1.0 has been dropped.}}
  • {{s4 K-Meleon 1.6 is now mostly supported (special builds required).}}
  • {{s5 Added support for Do-Not-Track header proposal (DNT), a filter like *$donottrack will cause the header to be sent to all locations (forum topic).}}
  • {{s6 Added an experimental "Recommend us on Facebook" button to the menu allowing easily recommending Adblock Plus — only displayed to Facebook users.}}

{{s7 User interface}}

  • {{s8 Malay and Estonian translations are included again.}}
  • {{s9 Allowed shortcut key preferences to have multiple values, the first available shortcut key is used.}} {{s10 Changed default Preferences key to Ctrl-Shift-E (used in Firefox 3.x), Ctrl-Shift-F (used in Firefox 4) or Ctrl-Shift-O (Firefox 4 on Mac OS X).}}
  • {{s11 Special icon state for "whitelisted" websites has been removed, the icon is now showing as "disabled" on such sites as well (forum topic).}}
  • {{s12 "Enable Adblock Plus" menu item has been renamed into "Disable everywhere".}}
  • {{s13 Firefox Mobile: First-run page is now more suitable for a small screen.}}
  • {{s14 Fixed: Contents of first-run page are sometimes cut off (forum topic).}}
  • {{s15 Fixed: "Edit filter in effect" in blockable items doesn't work correctly if Preferences dialog is already open.}}

{{s16 Issue reporter}}

  • {{s17 Disallowed reports with too many filter subscriptions.}}
  • {{s18 Disallowed reports for certain types of pages (in particular blank tabs, e-mails, local pages, extension pages).}}
  • {{s19 Thunderbird: Better reporting for blog posts, blog post address is now being sent instead of a meaningless mailbox address.}}
  • {{s20 Fixed: Disabled filter subscriptions should not be included in the report (forum topic).}}

{{s21 Core}}

  • {{s22 Implemented a faster filter matching algorithm (forum topic).}}
  • {{s23 patterns.ini file will only be saved on shutdown if some data actually changed.}}
  • {{s24 Matching and element hiding data are now cached in the Adblock Plus data directory to reduce startup time.}}
  • {{s25 Changed the way patterns.ini file is written to prevent data loss even if computer is switched off abruptly.}}
  • {{s26 Corrupted patterns.ini files are now recognized, a backup file is loaded automatically if available.}}
  • {{s27 Fixed: Element hiding rules might occasionally cause crashes (regression in Adblock Plus 1.3.3).}}
  • {{s28 Fixed: Disabling filters from the list of blockable items doesn't actually disable the filter in Firefox 3.6.}}
