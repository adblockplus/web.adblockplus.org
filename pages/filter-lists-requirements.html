title=Recommended filter list requirements description=A filter list must meet certain requirements in order to be recommended to users by Adblock Plus.

{{ filter-lists-requirements-intro[Filter Lists Requirements Intro] The list: }}

  • {{ filter-lists-requirements-item-1[Filter Lists Requirements List Item] Should be open source and have the possibility for users to report issues/suggest filters }}
  • {{ filter-lists-requirements-item-2[Filter Lists Requirements List Item] Must be unique (does not duplicate another list or filters in another list) }}
    • {{ filter-lists-requirements-item-2-example[Filter Lists Requirements List Item] Example: we will not support a second German filter list, or a list that has filters already contained in EasyList (i.e. do not create another "EasyList Germany" if it does not include additional or better filters) }}
  • {{ filter-lists-requirements-item-3[Filter Lists Requirements List Item] Must meet a certain quality }}
    • {{ filter-lists-requirements-item-3-example[Filter Lists Requirements List Item] Example: should have no redundancy, no extremely generic filters, no "slow" filters }}
  • {{ filter-lists-requirements-item-4[Filter Lists Requirements List Item] Must comply with certain minimum standards, as stated in the EasyList policy }}
  • {{ filter-lists-requirements-item-5[Filter Lists Requirements List Item] Must block only ads }}
    • {{ filter-lists-requirements-item-5-example[Filter Lists Requirements List Item] Example: it shouldnâ€™t block non-ads, i.e. content, or self-promotion, and it must not work around anti-ad block warnings and messages or technological measures that effectively control access to websites }}
  • {{ filter-lists-requirements-item-6[Filter Lists Requirements List Item] Must be actively maintained }}

{{ filter-lists-requirements-item-7[Filter Lists Requirements List Item] Finally, the filter list author must have a proficient understanding of the language he/she is creating the list for. }}
