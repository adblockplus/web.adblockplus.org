title=Detailed changelog for Adblock Plus 0.6.1

{{s1 The following lists the major changes compared to Adblock Plus 0.6:}}

{{s2 New features}}

  • {{s3 Filter subscriptions (formerly "synchronization")}}
  • {{s4 Sidebar can be detached (button in the upper-right corner)}}
  • {{s5 Showing the filter that has been applied in the sidebar's tooltip}}
  • {{s6 Double-click on a blocked/allowlisted item in the sidebar selects the filter in the Preferences dialog automatically}}
  • {{s7 Toolbar icon in Firefox (customizable — can be moved around or removed)}}
  • {{s8 Status bar icon has a context menu (same as the toolbar icon's menu)}}
  • {{s9 Middle-click on the icon in the toolbar or status bar disables/re-enables Adblock Plus}}
  • {{s10 Added minimal support for "DIV blocking" (see FAQ for details)}}

{{s11 User interface}}

  • {{s12 Added translations: German, Spanish, Italian, Russian, Polish, Slovak, Lithuanian, Romanian, Finnish, Turkish, Japanese}}
  • {{s13 Made Adblock Plus show up as an icon in the status bar (optional now, can be removed via Options)}}
  • {{s14 Mozilla Suite/Seamonkey: Added the sidebar and made it behave in the same way as in Firefox (opening via menu or Ctrl+Shift+B works)}}
  • {{s15 Removed limit for tooltip size in the sidebar}}
  • {{s16 Sidebar now tells that a page is allowlisted instead of "no blockable items"}}
  • {{s17 Sidebar always updating properly now even when going to a page without blockable items}}
  • {{s18 Whitelisting a site or a single page possible via toolbar icon's menu}}
  • {{s19 Preferences dialog: Filters are grouped by type}}
  • {{s20 Preferences dialog: Added a warning when Adblock Plus is disabled}}
  • {{s21 Preferences dialog: "Sort alphabetically" only sorts visually and any sorting applies immediately}}
  • {{s22 Preferences dialog: Sorting ignores non-alphabetical characters}}
  • {{s23 Preferences dialog: Multiple selection possible}}
  • {{s24 Preferences dialog: Reordering of filters via context menu or Ctrl-Up/Down (not when sorting is enabled)}}
  • {{s25 Preferences dialog: Blocked items in the suggestions list can also be selected now}}
  • {{s26 Preferences dialog: Using platform's line breaks for exported filters}}
  • {{s27 Preferences dialog: Added option to cancel the warning when importing filters}}
  • {{s28 Preferences dialog: OK doesn't perform "Add filter" automatically any more}}
  • {{s29 Preferences dialog: Moved parts of the "Options" menu into the new "Filters" menu}}
  • {{s30 Preferences dialog: Made menu items and OK button disable when necessary}}
  • {{s31 Preferences dialog: Options apply immediately now}}
  • {{s32 Preferences dialog: Line height decreased for the filters list and suggestions dropdown}}

{{s33 Core}}

  • {{s34 Made allowlisting apply to individual items as well, not only complete web pages}}
  • {{s35 Improved handling of frames (fixed: blocked frames wouldn't show up in the sidebar)}}
  • {{s36 When the option "block banner links" is used those links are treated as a separate element and can be blocked via context menu}}
  • {{s37 Optionally blocking ads in local pages now (enabled by default, some RSS readers need this)}}
  • {{s38 Uninstalling Adblock/Adblock Plus 0.5 automatically now (advising manual uninstall in Mozilla Suite and Seamonkey)}}
  • {{s39 Improved performance}}

{{s40 Known issues}}

  • {{s41 Flash/Java tabs aren't always visible and break the page's design in some rare cases}}
  • {{s42 Frames without blockable elements (including blocked frames) don't show up in the sidebar when going back to a page via history}}
  • {{s43 Tooltip for the toolbar icon is the same as for the status bar, isn't quite correct there}}
