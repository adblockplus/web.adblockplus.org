title=Detailed changelog for Adblock Plus 1.1

{{s1 The following lists the major changes compared to Adblock Plus 1.0.2:}}

{{s2 Known issues}}

  • {{s3 Thunderbird: When clicking "Subscribe" in the subscription list on first run, the entire Thunderbird window is closed. This should be fixed in Thunderbird soon (bug 503299).}}
  • {{s4 SeaMonkey: Toolbar icon looks ugly with the Modern theme. Modern theme doesn't seem to consider the possibility of extensions, using it with Adblock Plus isn't recommended.}}
  • {{s5 Some translations contain untranslated English strings.}}

{{s6 New features}}

  • {{s7 Added handling of HTTP redirects, the redirect address now shows up in the list of blockable items as well as the original address, both can be blocked.}}
  • {{s8 Added support for flexible anchors at start of filter (forum topic).}}
  • {{s9 Added support for ^ character that will match a single separator character or end of address (forum topic).}}
  • {{s10 Added support for domain exceptions in element hiding analogous to the $domain option of regular filters, e.g. "example.com,~foo.example.com##foo" or "~example.com##foo" (forum topic).}}

{{s11 User interface}}

  • {{s12 "Block" button on plug-ins (e.g. Flash or Java) is no longer shown permanently, it only shows up when the mouse is on the plug-in area.}}
  • {{s13 Added "Re-enable Adblock Plus on this page" context menu item for pages where Adblock Plus is disabled, this removes/disables the filter responsible.}}
  • {{s14 Browser integration code runs in its own context now, this makes conflicts with other extensions less likely.}}
  • {{s15 Changed the way "add subscription" links work, these can now only be triggered by user clicking a link.}}
  • {{s16 Added diagnostic page displaying all errors related to Adblock Plus as well as Adblock Plus version/source code revision, can be opened by typing chrome://adblockplus/content/errors.html into the location bar.}}
  • {{s17 Fixed: ABP icon updates its status every two seconds, hogs CPU time.}}
  • {{s18 Fixed: Detection of background images breaks the context menu.}}
  • {{s19 Fixed: "Disable on ..." doesn't work correctly if the exception rule already exists but is disabled.}}
  • {{s20 Fixed: "Disable on ..." doesn't work correctly with rules that already exist in filter subscriptions rather than user-defined filters.}}
  • {{s21 Fixed: Input focus issues when closing list of blockable items in current Firefox nightly builds.}}

{{s22 List of blockable items}}

  • {{s23 Added "document source" column, hidden by default (domain name to be used for the $domain filter option, forum topic).}}
  • {{s24 Added "Disable this filter on example.com" context menu item for user-defined filters (adds $domain option as required).}}
  • {{s25 Fixed: Creating a filter for an item matching a disabled exception rule wasn't possible again (forum topic).}}

{{s26 Preferences window}}

  • {{s27 "Slow filter" warning is now a separate list column which can be used for sorting.}}
  • {{s28 Using browser's real find bar now instead of faking it.}}
  • {{s29 "Close" button moved to the right side of the filter editor, made the button use Firefox 3 look.}}
  • {{s30 Improved position calculations for the filter editor, no longer makes the text position change.}}
  • {{s31 Fixed: Enabling/disabling many element hiding rules at once is slow.}}
  • {{s32 When a list of filters is pasted or imported, always insert comments before the filter that follows them (forum topic, forum topic).}}
  • {{s33 Fixed: Adding a filter subscription that redirects to a different address causes download failure.}}
  • {{s34 Fixed: Same filter might appear twice in the list in a particular scenario.}}
  • {{s35 Fixed: Wrong row is selected when the window is opened (worked around bug 489881).}}

{{s36 Filter composer}}

  • {{s37 Added flexible anchors as an option — "at the beginning of domain name" (forum topic).}}
  • {{s38 Added "example.com^" as fourth suggestion, block the entire domain (forum topic).}}
  • {{s39 Added warning when adding a filter to a disabled filter group (forum topic).}}

{{s40 Application support}}

  • {{s41 Dropped compatibility with old application versions and cleaned up code, now requiring at least: Firefox 3.0.9, SeaMonkey 2.0b1, Thunderbird 3.0b3, Songbird 1.2, Prism 1.0b1.}}
  • {{s42 Moved almost all application-specific code to the overlay file corresponding with that application, this should make application-specific changes easier in future.}}
  • {{s43 Fixed: Toolbar icon cannot be moved to a different location (SeaMonkey).}}
  • {{s44 Fixed: Toolbar icon might be added in one window but not another on first start (SeaMonkey and Thunderbird).}}
  • {{s45 Fixed: Indentation of some menu items of the toolbar icon's menu too large (Songbird).}}
  • {{s46 Fixed: Adblock Plus initialization fails (Prism).}}

{{s47 Core}}

  • {{s48 Added explicit support for blocking fonts (@font-type CSS directive in Firefox 3.5).}}
  • {{s49 Clearing browsing history in Firefox now clears hit statistics in Adblock Plus as well (this can be disabled with the hidden extensions.adblockplus.clearStatsOnHistoryPurge preference, forum topic).}}
  • {{s50 Cleaned up data management code, made sure Adblock Plus doesn't use up more memory than necessary.}}
  • {{s51 Fixed: Webpages can easily detect "Collapse blocked elements" option.}}
  • {{s52 Fixed: An error is reported to console on first run (patterns.ini not found).}}
  • {{s53 Fixed: Domain-restricted filters were applied to pages opened from disk.}}
