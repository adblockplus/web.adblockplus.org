title=Detailed changelog for Adblock Plus 0.7.5.4
  • {{s1 Default keyboard shortcuts have been changed to Ctrl+Shift+E (preferences) and Ctrl+Shift+V (blockable items) due to conflicts in Firefox 3 and Thunderbird (blog post)}}
  • {{s2 "Check banner links" option has been removed (forum thread), element hiding rules should be used as a replacement}}
  • {{s3 Whitelisting rules now apply to element hiding rules as well (bug 14814) (won't work properly in Firefox 3, see known issues)}}
  • {{s4 Hebrew (he-IL) and Malay (ms-MY) translations added}}
  • {{s5 Added support for new request types in Firefox 3 like XBL or XMLHttpRequest (documentation)}}
  • {{s6 Support for Songbird 0.4 and 0.5 added, now almost everything should work like in Firefox}}
  • {{s7 Songbird 0.2/0.3 support dropped}}
  • {{s8 Added support for eMusic Remote 1.0}}
  • {{s10 Support for Moblin browser added — untested}}
  • {{s11 Fixed filter editing issues caused by bug 372769 (Firefox 3 only)}}
  • {{s12 Fixed: Links in Adblock Plus dialogs don't open (Thunderbird only)}}
  • {{s13 Fixed: Input focus not put back to the filter list after editing a filter (Firefox 3 only)}}
  • {{s14 Fixed: The entry "No blockable items on current page" in the list of filter suggestions should not be selectable}}
  • {{s15 Fixed: Error messages when inserting filters in Preferences}}
  • {{s16 Fixed: Account for new XUL iframe behavior in Firefox 3}}
  • {{s17 Content policy now returns "blocked based on server" to trigger special handling in Firefox 3}}
  • {{s18 "Flash borders" in list of blockable elements will blink longer (forum thread)}}
  • {{s19 Fixed: Keyboard shortcut to close detached list of blockable items is not configurable}}
  • {{s20 K-Meleon: Done some major code refactoring}}
  • {{s21 K-Meleon: Added support for K-Meleon 1.5 (currently alpha)}}
  • {{s22 K-Meleon: Dropped support for K-Meleon 1.0}}
  • {{s23 K-Meleon: Status bar icon now reacts to middle-clicks and has a context menu (K-Meleon 1.5 only)}}

{{s24 Known issues}}

  • {{s25 Element hiding hits are not always registered in Firefox 3, this also prevents whitelisting rules to apply to element hiding (bug 425057)}}
  • {{s26 Songbird: Context menu doesn't always show up in Songbird, consequently not all items can be blocked via context menu}}
  • {{s27 eMusic Remote: Links in Adblock Plus dialogs load in current tab instead of opening a new one}}
