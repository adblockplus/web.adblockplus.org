title=Detailed changelog for Adblock Plus 0.7

{{s1 The following lists the major changes compared to Adblock Plus 0.6.1:}}

{{s2 New features}}

  • {{s3 Counting how often each filter gets applied — resulting hit count visible in the Preferences dialog}}
  • {{s4 Added possibility to disable individual filters, not only complete subscriptions}}
  • {{s5 Optimized processing of long filter lists, 5000 filters work without a noticeable slowdown (doesn't work for regular expressions)}}
  • {{s6 Vastly extended element hiding syntax — limiting by domain, attribute selectors, raw CSS}}
  • {{s7 Filter list now stored in adblockplus/patterns.ini in the profile directory instead of prefs.js}}

{{s8 User interface}}

  • {{s9 Added translations: Brazilian Portuguese, Dutch, Chinese Simplified, French, Danish, British English, Portuguese, Swedish, Chinese Traditional}}
  • {{s10 Preferences dialog: Made filter list a multi-column tree}}
  • {{s11 Preferences dialog: Added filter search (also find-as-you-type if enabled in browser)}}
  • {{s12 Preferences dialog: Filters and subscriptions can be reordered via drag and drop}}
  • {{s13 Preferences dialog: Added copy and paste for filters}}
  • {{s14 Preferences dialog: The list can be sorted by any column now}}
  • {{s15 Preferences dialog: Showing element hiding rules and invalid rules (mainly wrong regular expressions) in separate groups}}
  • {{s16 Preferences dialog: Element hiding rules are shown as invalid when used in a browser that doesn't support them (Gecko 1.7 and lower)}}
  • {{s17 Preferences dialog: Added an Edit menu}}
  • {{s18 Preferences dialog: Added Help button (opens FAQ)}}
  • {{s19 Sidebar: Added column sorting}}
  • {{s20 Sidebar: Can be filtered by address or type text}}
  • {{s21 Sidebar: Added filter and state columns (not shown by default)}}
  • {{s22 Sidebar: Tooltip showing more information now}}
  • {{s23 Sidebar: Images and background images are displayed in the tooltip for easier identifying (can be disabled via preference extensions.adblockplus.previewimages)}}
  • {{s24 Sidebar: Splitting up long addresses into multiple rows in the tooltip}}
  • {{s25 Sidebar: Current state is saved when detaching/reattaching}}
  • {{s26 Sidebar: Making sure to scroll to the item when flashing its borders (can be disabled via preference extensions.adblockplus.flash_scrolltoitem)}}
  • {{s27 Add subscription dialog: Made subscription title first input field}}
  • {{s28 Add subscription dialog: Added a dropdown list with a few suggestions for subscriptions}}
  • {{s29 Whitelisting via menu now allows to whitelist a specific page, not all its subpages as well}}
  • {{s30 Added an option to show/hide toolbar icon}}
  • {{s31 "Show in status bar" switched off by default}}
  • {{s32 Toolbar icon now visible in Mozilla and SeaMonkey in the personal toolbar}}
  • {{s33 Remembering now whether Preferences dialog/detached sidebar were maximized and restoring state when reopened}}
  • {{s34 Preferences extensions.adblockplus.defaulttoolbaraction and extensions.adblockplus.defaultstatusbaraction now define the action for left-click on toolbar/status bar icon}}
  • {{s35 Preferences extensions.adblockplus.sidebar_key, extensions.adblockplus.setting_key and extensions.adblockplus.enable_key now define the keyboard shortcut for opening/closing sidebar, opening preferences and enabling/disabling}}
  • {{s36 Selecting active locale on startup in Mozilla and SeaMonkey to fix installer's bad choice}}
  • {{s37 Fixed: Don't allow to add filters via "Disable on ..." while the Preferences dialog is open}}
  • {{s38 Fixed: Cannot open Preferences when either About or Subscription dialog is open}}
  • {{s39 Fixed: When switching to an already open Preferences dialog suggestion isn't filled in and the filter is not selected}}
  • {{s40 Fixed: Spaces shouldn't be stripped from subscription address/file name}}
  • {{s41 Fixed: Detached sidebar wouldn't save state in Firefox 1.0/Mozilla 1.7}}
  • {{s42 Allowed blocking background images via context menu}}
  • {{s43 Made it possible to drag images/links to the toolbar icon as well, not just the status bar}}

{{s44 Core}}

  • {{s45 Only exception rules starting with http:// or https:// will whitelist entire pages, other rules apply to individual items only}}
  • {{s46 Saving daily backups of patterns.ini (max. 5 backups by default)}}
  • {{s47 VERSION}} {{s48 in subscription addresses is replaced by current Adblock Plus version when updating, e.g.}} {{s49 ABP0.7}}
  • {{s50 Added interface IAdblockPlus to be used by other extensions (for example to manage external subscriptions)}}
  • {{s51 Moved several hardcoded lists to preferences}}
  • {{s52 Choosing which schemes can be blocked with a whitelist (extensions.adblockplus.whitelistschemes), not a blacklist}}
  • {{s53 Fixed: Blockable items list would show all visited pages, not just the current (mainly on Linux)}}
  • {{s54 Fixed: Memory leak happening with Gecko 1.8 (not Gecko 1.9) when "Show tabs on Flash and Java" is activated}}
  • {{s55 Fixed: Performance issues on pages with many occurences of the same address}}
  • {{s56 Fixed: Switching profiles at runtime in Mozilla and SeaMonkey isn't handled properly}}
  • {{s57 Fixed: Error message "}}{{s58 secureLookup(insecDoc, 'createEvent') is not a function}}{{s59 "}}
  • {{s60 Filters starting with http:// or https:// no longer match only at the beginning of the address}}
  • {{s61 Starting filters with | makes them match only at the beginning of the address, finishing them with | makes them match only at the end}}
  • {{s62 Fixed file permission problems when installing on Linux in Mozilla Suite/SeaMonkey}}
  • {{s63 Made sure there is no way for web sites to detect Adblock Plus}}

{{s64 Known issues}}

  • {{s65 Yahoo Application State plugin (npYState.dll, installed by Yahoo Messenger) causes crashes on Yahoo Mail if Adblock Plus is installed. It is suggested to remove this plugin, its only function seems to be to notify Yahoo Mail about your version of Yahoo Messenger.}}
  • {{s66 Flash plugin crashes when Mozilla ActiveX plugin is installed and the option "Show tabs on Flash and Java" in Adblock Plus is activated. It is suggested to deactivate this option.}}
