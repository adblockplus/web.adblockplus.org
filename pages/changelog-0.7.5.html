title=Detailed changelog for Adblock Plus 0.7.5

{{s1 The following lists the major changes compared to Adblock Plus 0.7.2:}}

{{s2 New features}}

  • {{s3 Matching element hiding rules are now visible in the list of blockable items}}
  • {{s4 Hit counts for element hiding rules}}
  • {{s5 Subscription authors can now specify the new address of the list using permanent redirects (bug 16786, documentation)}}
  • {{s6 Only Firefox 2.0.0.4/SeaMonkey 1.1.2 and higher: XMLHttpRequests and object subrequests appear in list of blockable items with type "other" (blog post)}}
  • {{s7 Compatibility with K-Meleon 1.1, multi-language packages are available now (those will not work correctly in K-Meleon 1.02)}}

{{s8 User interface}}

  • {{s9 Added translations: Persian, Arabic, Mongolian}}
  • {{s10 List of blockable items now appears at the bottom of the window instead of being placed in the sidebar, the behavior is now the same throughout host applications with exception of K-Meleon (forum thread)}}
  • {{s11 Appearance of tabs on Flash and Java changed to make them less intrusive (forum thread)}}
  • {{s12 Icons for the optional "State" column in blockable items list replaced to make them better recognizable for colorblind people (forum thread)}}
  • {{s13 Removed useless "Block remote ads in local pages" option (forum thread)}}
  • {{s14 Made sure tabs on Flash and Java do not break page layout (bug 12911)}}
  • {{s15 Tabs on Flash and Java now adapt to changes in text size}}
  • {{s16 Firefox: Default placement of the toolbar icon is now the right edge of the navigation toolbar (forum thread)}}

{{s17 Core}}

  • {{s18 Removed example filters added to the list by default, these have been superseded by subscriptions}}
  • {{s19 More consistent handling of frames, less likely to trigger bugs in Gecko (bug 16726)}}
  • {{s20 Added collapse and ~collapse filter options to override the global "Collapse blocked elements" option (documentation)}}
  • {{s21 Removed SeaMonkey sidebar customization hacks}}
  • {{s22 Removed several no longer useful preferences (documentation) and simplified the application logic}}
  • {{s23 Removed exceptions for about:blank, javascript: and data: addresses}}
  • {{s24 Now ignoring effectively empty rows/cols attributes when collapsing frames in a frameset}}
  • {{s25 Made handling of frames and images loaded with OBJECT and EMBED tags consistent (bug 16144)}}
  • {{s26 Dropped Songbird 0.1 support, requiring Songbird 0.2 or higher now}}
  • {{s27 Fixed: items with same address but different type always appear as one item in the list}}
  • {{s28 Fixed: "Open in new tab" and "Flash item's borders" context menu entries (blockable items list) don't work in Thunderbird and Songbird}}
  • {{s29 Fixed: Tabs on Flash and Java only visible after restarting twice after installation in Firefox 3.0 alphas (bug 16077)}}
  • {{s30 Now preventing web pages from probing for Adblock Plus by abusing bug 292789 in Gecko (bug 16027, documentation).}}
  • {{s31 Improved storage of internal data (bug 15758)}}

{{s32 Known issues}}

  • {{s33 Mac OS X: Flash movies might not be displayed properly if "Show tabs on Flash and Java" option is switched on}}
  • {{s34 Thunderbird/SeaMonkey Mail: Default keyboard shortcut Ctrl+Shift+A for Adblock Plus Preferences doesn't work due to a conflict with a built-in keyboard shortcut (bug 16791)}}
  • {{s35 SeaMonkey Mail and Songbird: No toolbar icon available (status bar icon shows up instead)}}
